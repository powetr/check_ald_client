(Актуализирована под ALD Pro 3.x)

Назначение: Автоматизированный аудит парка машин в домене ALD Pro.

Сбор данных о версиях клиентского ПО, ролях серверов и статусе подключения.

АКТУАЛЬНЫЕ ВЕРСИИ ALD PRO (СПРАВОЧНИК)

При анализе колонки CLIENT_VERSION используйте следующие ориентиры:

ALD Pro 3.2.0: АКТУАЛЬНАЯ ВЕРСИЯ (Мажорный релиз).

Базируется на обновленной платформе и Astra Linux 1.8.Рекомендуется для всех новых внедрений и планового перехода.

ALD Pro 2.4.x - 2.5.x: Поддерживаемые версии.

Предыдущая стабильная ветка. Хосты с этими версиями полностью функциональны,но требуют планирования обновления до ветки 3.x.

Версии ниже 2.4.0: УСТАРЕВШИЕ.

Подлежат обязательному обновлению. Использование данных версий может привести
к несовместимости с новыми подсистемами и снижению безопасности домена.

ТРЕБОВАНИЯ И ЗАВИСИМОСТИ

На стороне сервера (где запускается скрипт):

Активный билет Kerberos (kinit admin).

Утилита 'ipa' (пакет freeipa-adm-intf).

Утилита 'sshpass' (sudo apt install sshpass).

На стороне клиентов:

Разрешенный SSH-доступ для указанного пользователя.

Пакет aldpro-client для диагностики.

КЛЮЧИ ЗАПУСКА

-v Версия скрипта.

-l Режим списка (FQDN + IP). SSH не выполняется.

-n Clean mode: без заголовков и лишнего текста.

-S Silent mode: удаление команды запуска из истории (.bash_history).

-f <шаблон> Фильтр имен через регулярные выражения (grep -E).

-H <хост> Сканировать только ОДИН конкретный хост (FQDN).

-u <логин> SSH пользователь.

-p <пароль> SSH пароль (рекомендуется в одинарных кавычках '').

СОВЕТ: Для полной скрытости вводите ПРОБЕЛ перед командой запуска.

РАСШИФРОВКА СТАТУСОВ (ROLE_OR_STATE)

DOMAIN_CONTROLLER - Контроллер домена (основной или реплика).

SUB:[РОЛИ] - Сервер подсистемы (DHCP, DNS, REPO, SYNC и т.д.).

configured - Клиент корректно настроен и введен в домен.

unconfigured - Клиент установлен, но не присоединен к домену.

ENROLLED(manual) - Найден билет Kerberos (keytab), статус определен косвенно.

UNREACHABLE (N/A) - Хост недоступен по SSH.

ИСПРАВЛЕНИЕ СТАТУСОВ (ШПАРГАЛКА)

Ввод в домен:

aldpro-client-installer -u admin -p 'pass' -d domain.lan

Проверка времени (Kerberos):

ntpdate -u <IP_КД> или # systemctl restart chrony

ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

Список всех хостов: ./check_ald_clients.sh -l

Скрытый аудит: ./check_ald_clients.sh -S -u admin -p 'MyPass'
